Overview

How the platform works

Fanometrix Pulse has three core building blocks. Understanding how they relate makes everything else straightforward.

◫

Survey

The questionnaire — up to 4 questions with multiple-choice answers. A survey is a reusable template, not tied to any specific campaign or publisher. The same survey can run in dozens of campaigns simultaneously.

◎

Campaign

One survey running for one publisher, with specific dates, a response target, and a live/paused/closed status. The campaign controls whether responses are collected. Each campaign gets its own embed code.

⬡

Campaign Group

A bundle of campaigns served through a single embed code. When the embed loads, Fanometrix picks an eligible campaign and serves it. The publisher only ever needs one URL.

💡

Key principle: Surveys are templates. Campaigns are delivery vehicles. Campaign Groups are playlists. Publishers get one URL and you control everything else from the admin.

Section 2

Surveys

A survey is a questionnaire — up to 4 questions, each with 2–4 multiple-choice answers. Surveys are reusable templates, independent of campaigns.

Survey statuses +
StatusMeaning
DraftStill being written. Cannot be attached to a campaign yet.
ReadyComplete and approved. Can be attached to any campaign.
ArchivedHidden from the default view but still searchable and reusable.
DeletedSoft deleted. Not visible by default. Can still be restored by an admin.
ℹ️

A survey is never Live. Campaigns control whether responses are collected. The same survey can be used in a Live campaign, a Paused campaign, and a Closed campaign simultaneously.

Creating a survey +
1
Go to Surveys in the left navigation and click + Create Survey.
2
Give the survey a name and an optional description. Set status to Draft.
3
Add up to 4 questions, each with 2–4 answer options.
4
Customise the Thank You screen shown to fans after completing the survey.
5
Click Preview to see how it looks in the 300×250 MPU. No responses are recorded in preview mode.
6
When approved, change status to Ready. It can now be attached to campaigns.
Deletion rules +

A survey can only be deleted if it has zero responses and is not linked to any campaigns. If either condition is not met, the Delete button is disabled. Archive instead — archived surveys remain searchable and can be restored.

Section 3

Campaigns

A campaign connects a survey to a publisher. It controls when responses are collected, how many are targeted, and whether the embed is currently live.

Campaign lifecycle +
StatusMeaningResponses?
DraftBeing set up, not yet deployedNo
ScheduledReady but start date hasn't arrivedNo
LiveActively collecting responsesYes
PausedTemporarily stopped, can be resumedNo
ClosedPermanently finishedNo
ArchivedHidden from the default viewNo
⚡

Fanometrix automatically moves a campaign from Scheduled → Live when the start date arrives, and from Live → Closed when the end date passes or the target response count is reached.

Creating a campaign +
1
Go to Campaigns and click + Create Campaign.
2
Enter Brand Name and Campaign Name. Click Auto to generate the Campaign ID.
3
Set Start Date and End Date. The campaign goes live and closes automatically on these dates.
4
Optionally set a Target Responses number — the campaign closes automatically when this is hit.
5
Select the Survey and Publisher. Add the placement name.
6
Save as Draft. When ready to deploy, click Go Live Now or Publish.
Campaign card actions +

PublishDraftSets to Scheduled (waiting for start date)

Deletion rules +

Campaigns can only be deleted if their status is Draft or Scheduled AND they have zero responses. Campaigns with responses can never be hard-deleted — archive them instead. This protects reporting integrity.

Section 4

Campaign Groups

A Campaign Group bundles multiple campaigns together under one embed code. When the embed loads, Fanometrix picks an eligible campaign and serves it. Responses link to the specific campaign served.

When to use a Campaign Group +
  • You want to run multiple surveys on the same publisher placement at the same time
  • You want to rotate between surveys without the publisher updating their embed code
  • You want a single fallback if one campaign pauses or closes mid-flight
🏗️

The publisher gets one URL and trafficks it once. You control which surveys run, add new campaigns, pause individual ones, or close the whole group — without asking the publisher to change anything.

Rotation types +
TypeHow it worksBest for
EqualEach eligible campaign has the same chance on every loadMost situations — default
WeightedCampaigns with higher weight values are served more oftenWhen one survey should run more frequently
PriorityThe eligible campaign with the lowest priority number always serves firstOne primary survey with backups
Eligibility rules +

Every time the embed loads, Fanometrix checks each campaign in the group:

  • Group must be Live and within its date range
  • Campaign must have status Live
  • Campaign must be within its own start and end dates
  • Campaign must not have reached its target response count
  • Campaign's survey must exist and not be deleted or invalid

If no campaigns are eligible, the embed renders blank — the publisher sees an empty ad slot, not an error.

The embed code + 
/embed?group=carlsberg_research_wave1_fotmob&publisher=fotmob&placement=homepage-mpu&country=%%COUNTRY%%

Copy this directly from the group card in Campaign Groups. The publisher trafficks it once; you control everything else from the admin.

Section 6

User Management

Fanometrix accounts are organisation-based. Each account represents a company — a brand, agency or publisher — not an individual. No personal data (names, emails) is stored.

Account roles +
RoleWhat they can see
AdminEverything — full platform access including user management
BrandDashboard, Campaign Reports, Exports, Insights — for their allowed campaigns only
AgencyDashboard, Campaign Reports, Publisher Performance, Exports — for their allowed campaigns and publishers
PublisherDashboard, Publisher Performance — filtered to their own publisher data
Creating an account +
1
Go to User Management and click + New Account
2
Set a Username — letters, numbers, underscores and hyphens. Login is not case-sensitive.
3
Click ⚡ Generate Temporary Password. Copy the credentials — the password is shown only once.
4
Leave Force password change on first login ticked — the user sets their own permanent password.
5
Select Access Rights and enter the Organisation Name.
6
Set Campaign Access and Publisher Access to restrict the account, or leave blank for full access.
Password policy +
  • Passwords are stored as hashed values only — never visible to admins
  • Use Generate Temporary Password to create a strong random password
  • Share credentials via Teams, Slack, email or WhatsApp
  • The user sets their own permanent password on first login
  • Reset a password at any time via Edit → Generate Temporary Password

Demo data +

The Demo Data page lets you generate sample responses for any campaign. These are flagged as demo data and excluded from all reporting and response counts. Useful for testing that an embed is set up correctly. Demo responses can be deleted at any time without affecting real data.

Looker Studio integration +

The platform exposes a reporting view (vw_campaign_responses) that connects directly to Google Looker Studio via the PostgreSQL connector. See the Reporting page in the admin for connection details and API endpoint documentation.

📊

Responses, campaigns and surveys that have real data can never be hard-deleted, protecting your reporting and audit trail.

Common questions +

Can I use the same survey in two campaigns at once?

Yes. A survey is a template. You can attach it to as many campaigns as you like, all running simultaneously. Responses are separated by campaign in reporting.

What happens if all campaigns in a group are paused?

The embed renders blank — publishers see an empty ad slot. As soon as any campaign becomes eligible again, the embed starts serving automatically.

Can I change the survey attached to a live campaign?

Yes, by editing the campaign. It's usually safer to close the old campaign and create a new one to avoid mixing question sets in reporting.

How do I give a client access to their campaign data only?

When creating their account, use Campaign Access to select only the campaigns they should see. Their dashboard and reports are filtered automatically.
