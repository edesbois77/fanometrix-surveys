## 1. Executive summary

This release turns the product from a survey dashboard into a **multi-source research platform**. Its four pillars:

1. **Research Project Workspace v2** — a restructured, area-based workspace (Overview · Sources · Dashboard · Analysis · Reports · Conclusion & Knowledge) built on a shared data-provider layer.
2. **Conversation Intelligence** — a generic connector-based collection engine (YouTube live; News/Reddit/others pluggable), an append-only evidence lifecycle, two-stage AI relevance judging, a search-strategy planner, and an evidence-validation gate.
3. **Analysis & Reporting** — an aspect-organised synthesis workspace, a Research Plan advisor, contradictions/gaps detection, regeneration-safe researcher notes, and a portfolio of AI-generated reports.
4. **Research Library / Document Intelligence + Governance** — a page-by-page document extraction/vision pipeline and a document-governance permission model (ownership, visibility, learning, AI-access).

Plus a redesigned, cost-efficient **Survey Intelligence dashboard** and hardened **simulated-data isolation**.

## 3. User-visible changes

**Research Project Workspace**
- New area-based navigation: **Overview · Sources · Dashboard · Analysis · Reports · Conclusion & Knowledge**.
- Overview is now an executive snapshot with a **Next Recommended Action**; a persistent Activity utility route.
- Sources split into **Research** (choose a method) and **Execution** (operate collection).

**Conversation Intelligence**
- Create and run **Conversation Searches** with a research-first config form (Advanced options), relative time windows, and a **live YouTube connector**.
- **Collection run history** — each run is a reproducible, timestamped snapshot.
- **Review Evidence** in-project; the old Social Listening hand-offs are removed.
- **Search Strategy** planner turns a research question into a structured retrieval plan.
- **Evidence Validation gate** — only approved, included, relevant evidence feeds Analysis/Reports.

**Analysis & Reports**
- Aspect-organised **Analysis** workspace; each Research Aspect reads as a chapter with evidence-backed findings.
- **Contradictions & Research Gaps**; **Researcher Notes** that survive regeneration.
- **Research Plan** methodology advisor; Reports portfolio (Executive, Key Findings, Full Research Report, Article, Conclusion).

**Research Library**
- Upload **PDF + PNG/JPEG**; live **page-by-page** processing progress.
- Editable document metadata (title, author, description, tags, type, confidentiality) with per-field "manually edited" locks and an audit trail.
- **Document Governance**: owner, visibility (project/organisation/internal/platform), learning permission, AI-access, `nda_restricted` confidentiality; "Project Only" = bound to a single engagement.

**Survey Intelligence dashboard**
- Redesigned, denser layout: **collapsible filters**, a compact **Conversion Pipeline**, **Research Confidence** + **Collection Status** panels, insights surfaced high, four-question hierarchy.
- **User-controlled refresh** — loads once, then updates only on Refresh / filter / date / survey / campaign change / revisit. Shows **"Last refreshed: HH:MM:SS"**; non-destructive error state; no more auto-polling or per-second countdown.
- Dedicated Survey and Campaign scope filters; corrected KPI decimals.

**Campaigns**
- Bulk-edit selected campaigns; run-of-network default placement; friendlier deployment-card identity; not-Ready survey flagging; remembered list filters; origin-aware back navigation.

## 4. Architecture changes

- **Shared data layer** — `ProjectProvider` + `useResearchProject` centralise Research Project data; Product Walkthrough decoupled from the real workspace.
- **Generic Conversation Intelligence engine** — connector-agnostic collection (`collection_runs`, source-agnostic `social_mentions` columns, per-connector config in JSONB); YouTube live, others pluggable with no schema change.
- **Append-only Evidence Lifecycle** — one evidence base per search keyed by `(search_id, connector, external_id)`; `evidence_observations` ledger records every run encounter; `collection_runs` is the event ledger.
- **Two-stage collection** — retrieve → AI relevance-judge into structured evidence (score/confidence/rationale/research-aspect).
- **Analysis synthesis** — evidence grouped by AI-assigned Research Aspect; regeneration-safe human notes overlaid at read time.
- **Document extraction pipeline** — download → PDF text extract → per-page vision analysis (batched) → multi-stage document analysis, run in `after()` with `maxDuration=300`.
- **Document Governance model** — ownership/visibility/learning/AI-access as independent permission dimensions.
- **Middleware** — API routes now return JSON `401/403` instead of HTML redirects (security/UX hardening; no auth change).
- **AI provider** — OpenAI (`gpt-4o` / `gpt-4o-mini`) via shared `completeJSON` (bounded retry/backoff) and `classifyContent` wrappers.
- **Simulated-data isolation** — unscoped/platform-wide aggregates now filter `is_demo=false` / `is_simulated=false` at the application layer.

## 7. Known issues remaining (non-blocking)

**Observability / resilience** (from code review)
- `lib/ai-classify.ts` silently falls back to rule-based classification on OpenAI failure with no error log — an outage would be invisible. Add a `console.error`.
- No `fetch` timeouts (`AbortSignal.timeout`) on OpenAI/YouTube/Reddit calls — a hung upstream can tie up a function for the full `maxDuration`.

**Server compute** (from compute audit — Vercel Free-plan CPU pressure)
- Document extraction does **unbounded per-page** vision AI; collection runs make **one AI call per item** (Reddit item count uncapped). Consider caps / a background queue.
- Heavy AI routes without `maxDuration` (`full-research-report`, `research-projects` POST simulation, `generate-sample`, `evidence/generate`, `mentions/import`) risk timeout + retry double-billing.
- No caching anywhere; slow-changing stats/insights recompute every call.
- `/api/campaigns` GET performs per-row DB **writes** (status auto-transition) — should be a trigger/job.
- `/api/responses` still returns the full table unbounded (now demo-filtered, but not paginated).

**Cosmetic**
- Social-listening "coming soon" platform checkboxes remain selectable (inconsistent with the disabled treatment elsewhere).
- `/publisher-performance` is a navigable "Coming soon" page for publisher-role users — confirm that's intended at launch.

**Operational**
- **Free Supabase plan → no automated backups / no PITR.** The `pg_dump` custom-format dump (`fanometrix-prod-pre-106-125.dump`) is the only backup — keep an off-machine copy.
- Local git has two corrupted refs (`origin/main 2`, `stash 2`) — clean up before the merge/push.

## 8. Post-release recommendations

1. **Reduce server compute (highest priority for Free plan):** add `unstable_cache`/`revalidate` to stats/insights; stop `/api/campaigns` writing on GET; paginate `/api/responses`; set `maxDuration` + `AbortSignal.timeout` on heavy AI routes; cap document page-count and Reddit item-count. Then re-check Vercel usage before deciding on an upgrade.
2. **Address the two resilience items** (AI-classify logging; fetch timeouts).
3. **Adopt a frequent-merge workflow** — short-lived feature branches off `main`, merged and deleted per feature, to avoid another ~100-commit branch. (Git cleanup after this release creates the fresh baseline for it.)
4. **Standardise the release runbook:** because the DB runs ahead of code, always (a) back up via `pg_dump`, (b) run the migration state-check, (c) deploy code, (d) verify. This document is the template.
5. **Tidy cosmetics** (coming-soon checkboxes; confirm `/publisher-performance` intent).
6. **Watch AI spend** — the OpenAI-backed pipelines are the main variable cost; instrument per-route token usage.
